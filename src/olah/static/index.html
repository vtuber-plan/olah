Get Started in Three Steps

Set HF_ENDPOINT, keep your existing download workflow, and let Olah proxy and cache resources as they are requested.

1

Set the mirror endpoint

Use the current Olah server as your Hugging Face endpoint before you download anything.

2

Download with your usual tools

Python, CLI, models, or datasets all keep the same workflow. Only the endpoint changes.

3

Reuse the cached mirror

Olah stores fetched metadata and file blocks under your configured repos-path.

Linux / macOS Shell 
export HF_ENDPOINT={{scheme}}://{{netloc}}
Windows PowerShell PowerShell 
$env:HF_ENDPOINT = "{{scheme}}://{{netloc}}"
Python Example huggingface_hub 

from huggingface_hub import snapshot_download

snapshot_download(repo_id='Qwen/Qwen-7B', repo_type='model',
    local_dir='./model_dir', resume_download=True,
    max_workers=8)

Notes

A few operational details matter when you turn a local Olah instance into a mirror that other tools can depend on.

Current mirror URL

{{scheme}}://{{netloc}}

Where cached content lives

Downloaded metadata and file blocks are stored under your configured repos-path.

When host or port changes

Update mirror-netloc and mirror-lfs-netloc so clients receive correct mirror URLs.

Compatible workflow

You do not need to change repo IDs or download commands. In most cases, setting HF_ENDPOINT is enough.
